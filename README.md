

# home_credit_default_risk: ローン申請の大規模データにおける債務不履行予測モデルの構築と審査最適化
🔗 [Kaggle Competition Link]https://www.kaggle.com/c/home-credit-default-risk

<br>

## 1. プロジェクトの背景と目的
銀行口座を持たない人々に対し、安心で快適な融資体験を提供することで、金融包摂の拡大を目指しています。こうした金融サービスが行き届いていない人々が満足のいく融資体験を得られるよう、Home Creditは通信事業者情報や取引情報など、さまざまな代替データを活用して**顧客の返済能力を予測**しています。

<br>

* **データ提供元**:Home Credit Group（銀行口座を持たない人や信用履歴が少ない層に対し、POSローン（店頭分割払い）やアプリを用いた簡易ローンを提供）
* **主催**: Home Credit Group

## 3.アプローチの特徴  
1.FE設計における２本の柱(試走駆動、観点別)  
-試走駆動：Cleaningの段階で全７テーブルを集約・結合後にLGBM/CatBoostを試走して寄与度を確認し、それらに基づくEDA・FEを実施  
-観点別：試走結果で寄与度の高いEXT_SOURCE(外部から提供されたスコア)の幾何平均のFE時においては、欠損値をデフォルト率が近いスコアで補完。  
-返済予測に重要だと考えられる観点(返済能力・人物的評価・申告情報の整合性)に沿ってEDA・FEを実施  
 複数に分けてモデル試走を行い、追加した特徴量の中から高い寄与度を持つもののノイズとなる特徴量を検討し、削除を実施することで精度の低下を防止  
2.リーク防止を一貫したTarget Encoding設計  
-全てのTarget EncodingでOOF(Out-of-Fold) + Smoothingを適用し、テストデータへの情報漏洩を防止  
3.モデル実装後の分析・ビジネスプランの検討・API化  
-分析：SHAPを用いて、寄与度5%以上の相互作用を確認。閾値の検討  
-ビジネスプランの検討：一次審査通過者(モデル適用前データ)にスクリーニング的にモデルを適用。デフォルト判定時の対応が課題  
-API化：AWSにおいて、S3＋Lamda＋Gatewayを用いて、外部からモデルによる予測を可能とした
