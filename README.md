## 1. プロジェクトの概要と目的  
**【背景】**  
金融サービスが行き届いていない人々が満足のいく融資体験を得られるよう、Home Creditは通信事業者情報や取引情報など、さまざまな代替データを活用して**顧客の返済能力を予測**しています。  
(Kaggle抜粋)
<br>
* **データ提供元**:Home Credit Group（POSローン（店頭分割払い）やアプリを用いた簡易ローンを提供）
* **主催**: Home Credit Group
<br>

**【目的】**  
本プロジェクトでは、ローン審査における顧客の返済能力を予測し、企業方針(信用情報が薄い層へのローン提供)を維持しつつ、利益改善を目的とする。
データの制約(融資申請却下された人のデータ欠如)から全自動判定は当該コンペにおいて不可能であるため、現状の審査に追加スクリーニングとしてモデル判定を実施。
デフォルトリスクが低い層を追加承認し、リスクが高い層は人間による2次審査に移行することで利益改善と企業方針を両立させる。

## 3.ビジネスプラン：審査最適化の設計  
<pre>
【業務処理フロー】
1次審査通過者(現状の審査)  
　　│  
　　▼  
  モデルスコアリング（予測確率算出）  
  　│  
　　├─ 予測確率 < 0.23 ──→ 自動承認（全体の約85%）  
　　│  
　　└─ 予測確率 ≥ 0.23 ──→ 2次審査（人間による審査、約15%）  
　　　　　　　　　　　　　　　　　│  
　　　　　　　　　　　　　　　　　├─ 承認  
　　　　　　　　　　　　　　　　　└─ 却下  
</pre>
【閾値設計（意思決定の根拠）】  
精度・利益・承認率のトレードオフを可視化し、最適な意思決定を検討。  
<img src="https://github.com/user-attachments/assets/518a8dd8-67d6-4008-af86-b94e755c6288" width="950">

【最適な閾値、正常Precision】  
- 閾値0.23で利益改善率が最大（+22.3%）
- 承認率85%を維持し業務効率を確保
- 正常Precision 95%以上を確保
- 閾値0.17～0.29の範囲で利益改善率が+20%以上であり、ロバスト  
以上より、精度・利益・業務効率のバランスが最も優れる閾値0.23を採用。  
なお、貸倒検知精度は26.5%のため、デフォルト判定された申請については2次審査に移行。

<br>
<pre>
■ 純改善額の算出  
  1. ベースライン（モデルなし：全件承認）  
     = 全正常顧客の融資額 × 利益率(0.1) − 全デフォルトの融資額 × LGD(0.6)  
  ※補足　LGD:「デフォルト時損失率」（Loss Given Default)  

  3. モデル導入時  
     = TN（正しく承認した正常顧客）の融資額 × 利益率(0.1)  
       − FN（見逃しデフォルト）の融資額 × LGD(0.6)  
       − (TP + FP)*1000(審査コスト)  

  5. 純改善額 = モデル導入時 − ベースライン  

【パラメータ設定】  
LGD = 0.60　　　　　　　　　　　 # デフォルト時の損失率（60%）  
利益率 = 0.10   　　　　　　　   # 完済時の利益率（10%）  
審査コスト = (TP + FP)*1000     # 2次審査移行した件数に審査単価を乗じた金額（通貨単位不明のため仮値）  
※実測値不明につき、上記の係数等は仮決め値。  
</pre>
■ 前提条件  
業務処理フロー上でデフォルト判定(TP,FP)したものは  
人間による2次審査に移行するため、審査コストを除く利益や損失は計算式から除外。  
  - TP（検知デフォルト）は2次審査で却下される想定  
  　実際には2次審査でも見逃しが発生するため、上記の純改善額の減少の要因となる。  
  - FP（誤検知した正常顧客）の2次審査で融資される想定  
    実際には2次審査で承認されるケースがあるため、機会損失は過大に見積もられている。  
この2つは逆方向に相殺しあうため、影響はある程度限定的になると思料されるが、相殺の程度は人間審査精度に依存するので、運用開始後に検証を要す。  
閾値0.23では貸倒検知精度(TP)は26.5%であるため、2次審査の精度がそれ以上であれば利益率は上記より増加が見込み。

## 5.主な発見と学び

1.EDA・FE全体  
　-寄与度の高い特徴量(EXT_SOURCE)同士は情報が重複し、合成しても精度向上に繋がらない場合がある。  
 またカテゴリ型の特徴量の組み合わせでは細分化しやすく、サンプル不足による過学習を招くため、Coverage（被覆率）の確認が必要。  
2.追加特徴量  
【FE1で追加した特徴量】  
・EXT_SOURCESの平均、幾何平均がFE1追加時点の試走時点で寄与度が支配的になった。LGBMでは幾何平均の方が寄与度が高く  
　CATBoostではその逆であった点は興味深く、カテゴリ型に強いCATBoostとの違いであると考えられる。  
  SHAP値を見ると、幾何平均の方がSHAP値のレンジが広く、単独で強く特徴を捉えているのに対して  
  平均では相互作用との傾向が一致しており、この点で相互対象がカテゴリ型であるため、CABoostでは  
  高く評価されたものと思われる。

【FE2で追加した特徴量のうち寄与度1%以上】
・CREDIT_ANNUITY_RATIO(完済年数) 2.38%(FE2追加後の試走)
　完済年数の寄与度が高い点は自然ではあるが、借入額と年収比よりも上位に来てる点は意外であった。
　時間的経過のリスクは金銭面だけでなく、健康や本人の性向の変化など、様々なリスクを孕んでいると思われる。
・GEOM_MEAN_RISK (職業・職種・学歴・インカムタイプのデフォルト率の幾何平均) 1.92%(FE2追加後の試走)
  それぞれを単純結合しただけでは細分化による過学習となるが、デフォルト率を用いて加工することで有効な特徴として組込みやすくなった。
・DIFF_GOODS_CREDIT(商品価格-借入額) 1.61%(FE2追加後の試走)
　商品の本体価格以外の付随費用の可能性もあるが、差額が大きいものは商品購入自体が目的ではない可能性も考えられうる。

## 6. 制約と今後の課題
【データ上の制約】  
・本データは融資実行済みの顧客のみで構成されており、審査段階で却下された申請は含まれない（サバイバーシップバイアス）。1次審査の自動化を行うためには却下データを含めた再学習・構築修正が必要  
・通貨単位が不明のため、コスト試算は相対比較での提示に留まる。パラメータを実データに置き換えた際の再試算が必須  
・単一時点のデータであり、時系列的な変動（景気変動、季節性）を捉えていない。本番運用ではモデルドリフトの監視が必要  
【モデル上のリスク】  
・SHAP寄与度上位をEXT_SOURCE由来の特徴量が占めており、外部スコアの提供元が変更・停止した場合の影響が大きい  
・ORGANIZATION_TYPE（業種）はSHAP分析でばらつきが大きく、サンプル数の偏りによる過学習リスクがある  
【今後の改善余地】  
・bureau_balance以外のサブテーブルの集約が一律min/max/mean/sum/std/latestに留まっており、ここの深堀で精度向上が見込める
・ノイズとなるカラムの検討・削除
・XGBoost,NN追加及びアンサンブル
・Stacking
