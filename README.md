

# home_credit_default_risk: ローン申請の大規模データにおける債務不履行予測モデルの構築と審査最適化
🔗 [Kaggle Competition Link]https://www.kaggle.com/c/home-credit-default-risk

<br>

## 1. プロジェクトの背景と目的
銀行口座を持たない人々に対し、安心で快適な融資体験を提供することで、金融包摂の拡大を目指しています。こうした金融サービスが行き届いていない人々が満足のいく融資体験を得られるよう、Home Creditは通信事業者情報や取引情報など、さまざまな代替データを活用して**顧客の返済能力を予測**しています。

<br>

* **データ提供元**:Home Credit Group（銀行口座を持たない人や信用履歴が少ない層に対し、POSローン（店頭分割払い）やアプリを用いた簡易ローンを提供）
* **主催**: Home Credit Group

## 3.アプローチの特徴  

1.FE設計における２本の柱(試走駆動、観点別)  
　-モデル試走結果の寄与度と観点別(返済能力・人物的評価・申告情報の整合性)の２軸に沿ってEDA・FEを実施  
2.リーク防止を一貫したTarget Encoding設計  
　-全てのTarget EncodingでOOF(Out-of-Fold) + Smoothingを適用し、テストデータへの情報漏洩を防止  
3.モデル実装後の分析・ビジネスプランの検討・API化  
　-SHAP分析→閾値設計→運用フロー検討+コスト試算→AWS S3 + Lambda + API Gateway でデプロイ

## 4.パイプライン概要
| 項目 | 内容 |
| :--- | :--- |
| **01_Cleaning** | ダウンキャスト(メモリ最適化)、サブテーブル集約、テーブルマージ、モデル試走、リーク検討、Parquet出力(zstd) |
| **02_EDA1** | 不均衡度確認、欠損・異常値分析、ターゲット変数の索敵、試走寄与度に基づく第1・第2有力指標の分析、境界値検出 |
| **02_EDA2** | 返済能力・人物評価・申告整合性の観点別分析、その他の組み合わせ検討 |
| **03_FE1** | EXT_SOURCE合成特徴量(平均・幾何平均・境界値フラグ)、業種別TE、ビン別TE、効果検証、モデル試走、不要追加カラム削除、Parquet出力(zstd)|
| **03_FE2** | 返済能力、人物評価、申告整合性、その他組み合わせ特徴量の過学習検証・削除 |
| **04_Model(CPU)** | Optuna(サンプリング)、LGBM+CatBoost StratifiedKFold、アンサンブル重み最適化 |
| **04_Model(GPU)** | Optuna(フルデータ)、LGBM+CatBoost本番学習、Kaggle提出結果の比較(CPU,GPU) |
| **05_Analyze** | 寄与度分析、アンサンブルSHAP、混同行列、閾値検討、自動承認率の検討、アンサンブル配分AUC可視化 |
| **06_Business** | 運用フロー設計、コスト試算、閾値最適化、リスクと限界の整理 |
| **07_Deploy** | サンプル推論テスト、GitHub Actions→ECR→Lambda CI/CD、API Gateway構築 |
